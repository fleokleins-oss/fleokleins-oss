# Ação Imediata – 30 Minutos Para Sair do Zero

Este arquivo é um guia enxuto para você começar agora.  Siga os passos na ordem e marque quando terminar.  Em meia hora você terá a base dos seus perfis pronta para enviar propostas.

## ✅ Checklist de 30 minutos

1. **Abrir o GitHub** e criar o repositório **fleokleins-oss** (caso ainda não exista).  Adicione um `README.md` com o conteúdo de `GITHUB_README_REAL.md`.
2. **Criar o repositório `etl-postgres-docker`.** No navegador mesmo, crie o repositório e faça upload dos arquivos modelo: `docker-compose.yml`, `src/etl.py`, `requirements.txt` e um `README.md` simples (veja `README_etl_repo.md` para referência).  Não precisa do Airflow; use o script ETL básico.
3. **Criar o repositório `sql-metrics`.** Crie a pasta `sql/` e coloque algumas queries úteis (você pode copiar as da pasta `examples/` ou escrever as suas).
4. **Atualizar o LinkedIn**: copie o texto de `LINKEDIN_REAL.md` para a seção **Sobre**, crie uma experiência “Data Engineer (Freelancer)” e adicione 10 skills (Python, SQL, PostgreSQL, Docker, etc.).
5. **Atualizar o Workana**: ajuste o título para “Data Engineer | Python, SQL, PostgreSQL, Docker”, adicione as 4 habilidades principais, copie a visão geral e inclua links do GitHub/LinkedIn.
6. **Enviar 1 proposta** para um job de ETL simples.  Use o template de proposta e ofereça entrega em 48–72 h.

Terminou a lista? Parabéns!  Agora siga o roteiro de 7 dias no `WORKANA_REAL.md` para se aprofundar.
